# Harness manifest

Source of truth for agents, commands, and skills is **`.harness/`** ‚Äî the
`harness` CLI generates the `.claude/` stubs and copies. Never hand-edit
generated stubs (they carry a GENERATED banner); edit the `.harness/` source
and run `harness stubs` (agents/commands) or `harness doctor --fix` (everything).
Machine-readable config: `.harness/harness.json`.

## Agents ‚Äî source `.harness/agents/`, stubs `.claude/agents/`

| Agent        | Model  | Use when                                                                                             |
| ------------ | ------ | ---------------------------------------------------------------------------------------------------- |
| `indexer`    | haiku  | Project index is stale or missing ‚Äî also via `/index`                                                |
| `researcher` | sonnet | A question needs investigating before building (VS Code API, Claude Code internals, Copilot interop) |
| `developer`  | opus   | Implementing features, fixes, refactors, tests                                                       |
| `reviewer`   | opus   | Reviewing a diff or branch before merge                                                              |

## Commands ‚Äî source `.harness/commands/`, stubs `.claude/commands/`

- `/index` ‚Äî dispatch the Indexer to rebuild the project index.
- `/approve-memories` ‚Äî review pending team-memory proposals (CLI equivalent: `harness approve`).

## Skills ‚Äî source `.harness/skills/`, copies `.claude/skills/`

- `vscode-extension-dev` ‚Äî extension architecture rules; fires on activation/contributes/bundling/testing work.
- `webview-ui` ‚Äî CRM webview panel rules; fires on webview/CSP/messaging work.
- `agent-runtime-interop` ‚Äî Claude Code & Copilot integration knowledge; fires on interop work.
- `harness-manifest` (user-invoked) ‚Äî this map.

## Everything else

- **Memory**: protocol `.harness/protocol/memory-protocol.md`; ground rules `.harness/protocol/ground-rules.md`; personal `.harness/memories/`, team `.harness/team-memories/team.md` (+ `.pending/`), index `.harness/project-index/`, marker `.harness/last-index-commit`.
- **Hooks** (husky, `.husky/` ‚Äî supersedes `harness init`'s `.git/hooks` install): `commit-msg` commitlint (Conventional Commits), `pre-commit` lint-staged (no-op until `tsconfig.json` exists), `pre-push` tests + staleness check, `post-merge` staleness check + Copilot sync via `.harness/scripts/`.
- **Copilot bridge**: `.github/copilot-instructions.md`, regenerated by `npm run harness:sync-copilot`.
- **Dashboards**: project dashboard `npm run harness:dashboard` (edits `.harness/` sources, regenerates stubs on save); multi-project `harness ui`.
- **Drift check**: `harness doctor` (add `--fix` to repair stubs/copies).

When amending the harness: edit the `.harness/` source, update `harness.json` if the roster changed, run `harness doctor --fix`, and update this file ‚Äî all of it or the edit is incomplete.
